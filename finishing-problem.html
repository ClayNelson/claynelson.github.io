The caddie principle

Every shot in golf has a small chance of going wrong. Play 18 holes alone and small errors compound into a disaster round. Play with a caddie who checks each decision before you swing and most errors get corrected before they cascade.

That's the verification gate. Not a better golfer. A checkpoint that catches mistakes before they become the next hole's problem.

Cursor · Windsurf · Claude Code

"We will build the perfect golfer."

Better model → fewer mistakes → verification not needed. The path to dominance is model quality and IDE experience.

In regulated industries, "fewer mistakes" isn't the bar. Zero undetected mistakes, with provenance, is the bar. ISO 26262 doesn't accept probability.
GitHub

"We will build the best caddie system."

Freedom of models — use any golfer you want. But every swing goes through a verification loop running on Actions, signed to GitHub's audit trail.

The caddie system improves regardless of which model is on top. Model wars are someone else's problem.
Why this matters now: The model quality gap is closing fast. But you cannot replicate 38 years of Parasoft MISRA certification, UseBlocks' ISO 26262 requirement tooling, or the Actions ecosystem of 25,000+ integrations with a better model. The moat isn't the golfer. It's the infrastructure the caddies run on.

Network density — the caddie marketplace
25,000+ Actions. Every safety tool vendor already has a GitHub integration or is building one. Cursor has zero. Claude Code has zero.
They have no compute event infrastructure. Running Parasoft requires triggered compute, artifact storage, SARIF reporting — not just a chat interface.
2
Causality — GitHub sees every commit, not snapshots
GitHub holds the full graph: who wrote it, what requirement triggered it, which test passed it, which agent touched it.
You can't reconstruct causality after the fact. It has to be captured at commit time, in the platform where commits live.
3
Audit trail — compliance as a byproduct
SARIF to Security tab. SBOM signed with Sigstore. Requirements linked to commits. All in GitHub — the system of record auditors already ask for.
An IDE tool can't be a compliance system of record. Regulated buyers won't accept "trust the chat history."
4
Model freedom — the caddie works with any golfer
Copilot runs GPT-4o, Claude, Gemini, or a customer's fine-tuned model. Cursor is betting on one stack. If the model changes, their moat evaporates.
In regulated industries, customers can't be locked to one model provider for liability reasons. Freedom-of-models is a compliance feature.
⛳
The bet our competitors are making

Cursor, Windsurf, and Claude Code are racing to build the perfect golfer — a model so accurate it doesn't need verification. That's a reasonable bet for a startup building a code editor.

📐
Why it breaks in regulated software

ISO 26262 doesn't accept "our model was 99.9% accurate." It requires documented evidence that every requirement was traced, every violation was caught, every artifact was signed. A perfect golfer who keeps no scorecard can't win a tournament with judges.

🏗️
What GitHub built that they didn't — Actions

Actions is the event infrastructure that makes verification gates possible at scale. Every push, PR, and merge is a programmable event. Parasoft fires on every PR. UseBlocks validates every commit. InfoMagnus signs every artifact. Automatically. Without a human remembering to run it.

🤝
The partner ecosystem is the caddie corps

Parasoft has 38 years of MISRA certification. UseBlocks owns ISO 26262 requirements tooling. They built for GitHub because GitHub has the platform — the compute, the events, the artifact storage, the security tab. Cursor can't offer them a home.

🔒
The causality moat

GitHub holds the full history: which requirement drove this commit, which agent touched this file, which test verified this function. Competitors see the code. GitHub sees the causality chain behind it.

🆓
Freedom of models makes the moat model-proof

Cursor is betting their moat on one model stack. When the next model leapfrogs it, their moat moves. GitHub's moat doesn't care who the golfer is — the caddie system works with any model.
