Nodes vs. edges: what agents lose without the context graph

Any AI agent can read the assets in a codebase. The question is whether the evidence connecting them is stored somewhere — and whether that record can survive a regulatory audit.

GitHub holds the evidence: WHY the code was written (ISO §4.5), WHAT constrains it (ADR-014), WHAT validates it (TC-042), WHAT gates it (compliance v2.4), and WHAT history prompted it (Issue #1337 → PR #892). An AI agent reads this graph — it does not replace it. The record is what survives an audit.
Six related assets. Any agent can read every one of them in isolation. It can reason about the code's logic. But the connections between them — the requirement it satisfies, the field failure that prompted it, the compliance gate it feeds — are not stored anywhere. When an auditor asks for the evidence chain, there is nothing to produce. Reasoning without a record.
The argument

Most platforms in the software development lifecycle hold snapshots. GitHub holds causality. Every commit carries a thread: who changed it, when, what issue it was responding to, what decision preceded it, what test it broke or fixed.

Layered over years of a software project, this becomes something a regulated industry cannot afford to lose — an organizational memory that is also a legal record. Not just what the software does, but why it became what it is, with the timestamps and attribution to prove it.

The question an auditor asks under ISO 26262, DO-178C, or IEC 62304 is not "did you run static analysis?" It is: can you produce the evidence that this version of this module was analyzed under these requirements, that the result was a gating condition on this release, and that no step in that chain was altered after the fact?

A legally defensible evidence chain has four properties an AI agent cannot supply on its own:

  • Immutability. Git's SHA-based commit graph is cryptographically tamper-evident. An agent's inference about what probably happened is not.
  • Attribution. Every change must be traceable to a named, authorized human. Git carries this natively. A reconstructed graph assembled from inference does not.
  • Timestamp integrity. The when must be verifiable. Commit timestamps are part of the hash chain. Anything assembled downstream is derived, not authoritative.
  • Completeness of the causal chain. Requirement → Design Decision → Implementation → Test → Gate → Release. GitHub is the only single system where all six exist in one place, connected by native references.

Any agent can reason about your code. Only one system holds the evidence chain that makes that reasoning auditable.

The AI model is the reader. GitHub is the record. In regulated industries, that distinction is not architectural preference — it is the difference between passing an audit and failing one.
